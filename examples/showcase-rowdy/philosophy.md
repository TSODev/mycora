# Philosophy

Rowdy's whole pitch is in one README line: built "for developers, DBAs,
and terminal enthusiasts who want to inspect, query, and manage their
databases without ever leaving their terminal or touching a mouse." It
compiles into a single standalone binary with no runtime dependencies —
the same terminal-first, no-GUI-required instinct behind every TSODev
tool.

- [[One interface, six database engines]] — SQL, key-value, and document
  databases behind three shared traits
- [[Single binary, no runtime dependencies]] — what that promise
  actually costs to keep

Where Mycora's bet is tree-plus-links for notes, Rowdy's bet is that a
terminal client for six wildly different database engines (relational,
key-value, document) doesn't need six different tools, or even six
different UIs bolted together — just one connector abstraction
underneath a consistent set of screens.
